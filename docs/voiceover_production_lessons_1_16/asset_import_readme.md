# Asset Import README — Lesson Voiceover Handoff

## Purpose

This document tells the import operator how to receive, verify, and import
approved human-voiceover recordings into the app asset tree for Lessons 01–08
(Batch A and Batch B).

---

## What Is In This Handoff Package

| File | Purpose |
| --- | --- |
| `human_import_handoff_sheet.csv` | One row per approved segment. The import operator works from this file. |
| `human_import_handoff_summary.md` | Counts and audit of what is in scope and what was excluded. |
| `asset_import_readme.md` | This file. |

The handoff sheet was generated by:

```powershell
dart run tool/export_human_import_handoff.dart
```

It can be regenerated at any time. See [How To Regenerate](#how-to-regenerate)
below.

---

## Scope

**In scope (QC_PASS, import_status = READY_FOR_IMPORT):**

| Batch | Lessons | Row count |
| --- | --- | --- |
| BATCH_A | L01 Arabic Starts Here, L02 First Real Word Success | 37 |
| BATCH_B | L05 One More Real Word, L06 First Fixed Expression, L07 Hear What You Already Know, L08 Your First Usable Arabic Pack | 81 |

**Excluded from this handoff:**

- Lesson 03 (`REVISE_REQUIRED`): connected-shape lesson requires script revision before recording.
- Lessons 04, 09, 10, 11, 12 (`NATIVE_REVIEW_REQUIRED`): held pending native-speaker audio review.
- Lessons 13–16 (`PLACEHOLDER_ONLY`): not yet scripted.
- Any individual segment with `export_state=HOLD`: orthographic fragments and
  build artifacts that are not natural continuous speech. These must be cleared
  manually before they can enter any recording or import step.

## Handoff Sheet Column Reference

| Column | Description |
| --- | --- |
| `lesson_id` | Stable lesson identifier. Matches the source lesson data. |
| `lesson_title` | Human-readable lesson title. |
| `segment_id` | Stable per-segment identifier (e.g. `L01_SEG_001`, `L01_AR_001`). |
| `segment_type` | Role of the segment: `opening`, `section_header`, `teaching_narration`, `instruction_prompt`, `word`, `list`, `short_phrase`, etc. |
| `approved_audio_filename` | The actual delivered filename as returned by the recorder. Blank in task-sheet mode; filled after QC intake. |
| `planned_audio_filename` | The stable logical target filename the app uses (e.g. `l01_ord_001_normal.mp3`). Import must place the file at `target_asset_path` under this name. |
| `batch_id` | Which recording batch this segment belongs to: `BATCH_A` or `BATCH_B`. |
| `qc_status` | `QC_PASS` for all rows in this handoff. |
| `import_status` | Operator-managed status. See [Import Status Values](#import-status-values) below. |
| `target_asset_path` | Relative path within the project where the final `.mp3` file must land (e.g. `lesson_01/voiceover/l01_ord_001_normal.mp3`). |
| `script_text_ar` | Arabic text for this segment (if any). Used to verify the delivered audio matches the script. |
| `script_text_support` | English narration or support text. Used to verify the delivered audio matches the script. |
| `target_duration_sec` | Target recording duration in seconds. Use for sanity-checking delivered audio length. |
| `delivery_note` | Performance note for the recorder. Import operator can use this to verify delivery style. |
| `final_resolution` | Current resolution state from QC. `READY_FOR_IMPORT` for all rows in this handoff. |
| `notes` | Pipeline notes. May contain review flags or clarifications. Generally blank for READY_FOR_IMPORT rows. |

---

## Import Status Values

| Value | Meaning |
| --- | --- |
| `READY_FOR_IMPORT` | Segment is approved and waiting for import. No blockers. |
| `HOLD` | Segment is from an approved lesson but is individually held (e.g. review flag not yet cleared). Do not import until the hold is lifted. |
| `BLOCKED` | Upstream dependency prevents import. Wait for upstream resolution. |
| `IMPORTED` | Import operator marks this once the file has been placed at `target_asset_path` and verified. |

## Import Steps

### 1. Prepare the delivery folder

Collect all delivered `.wav`, `.m4a`, or `.mp3` files from the recorder.
Organize them so each file's name matches the `approved_audio_filename` column
(or the `planned_audio_filename` if the recorder used the planned names directly).

### 2. Run a dry run

```powershell
dart run tool/import_human_audio.dart `
  --delivery-root="<path_to_delivery_folder>" `
  --revision=20260321-batch-a `
  --dry-run
```

Review the dry-run output to confirm:
- All expected files were discovered.
- No unexpected stem mismatches.
- No duplicate files for the same target stem.

### 3. Run the actual import

```powershell
dart run tool/import_human_audio.dart `
  --delivery-root="<path_to_delivery_folder>" `
  --revision=20260321-batch-a
```

The importer will:
- Copy or transcode each file to the correct `target_asset_path` location.
- Append the revision suffix (`__human__20260321-batch-a.mp3`) to preserve co-existence with the AI version.
- Update `assets/data/audio_manifest.json` to register the human variant.

### 4. Mark rows as IMPORTED

After a successful import, update the `import_status` column in
`human_import_handoff_sheet.csv` from `READY_FOR_IMPORT` to `IMPORTED` for
each segment that was successfully placed.

### 5. Verify in-app routing

Run the app and exercise the imported lessons. Confirm the voiceover preference
switch correctly routes to the human varianst where expected.

## Filename Convention

Every delivered file must map to a target path of the form:

```
assets/audio/lesson_<NN>/voiceover/<stem>_normal.mp3
```

Where `<stem>` matches the `asset_stem` value from the recording task sheet
(e.g. `l01_ord_001`, `l01_ar_001`).

The import tool appends the revision suffix. The app reads from the manifest to
select between `ai` and `human` variants at runtime.

Full convention spec: `docs/voiceover_production_lessons_1_16/audio_filename_convention_spec.md`

## What To Do With Held Rows

Segments with `import_status=HOLD` must not be imported until the hold is
explicitly lifted. To lift a hold:

1. Identify the hold reason from the `notes` column or from the review queue
   in `recording_intake_qc_outputs/retake_queue.csv`.
2. Resolve the issue (re-record, revise script, confirm spoken form).
3. Re-run the intake update script with the corrected intake manifest.
4. Re-run the handoff export to regenerate the sheet.

---

## What To Do When New QC Results Arrive

Once the QC intake layer has been run and recordings have been checked, regenerate
the handoff from the intake outputs:

```powershell
dart run tool/update_recording_status_from_intake.dart `
  --intake-manifest=<path_to_filled_intake_manifest>

dart run tool/export_human_import_handoff.dart `
  --intake-manifest=docs/voiceover_production_lessons_1_16/recording_intake_qc_outputs/completed_segments_report.csv
```

This will:
- Re-filter to only rows where `qc_status=QC_PASS` from the actual intake.
- Set `approved_audio_filename` from the `received_filename` in the intake manifest.
- Regenerate both `human_import_handoff_sheet.csv` and `human_import_handoff_summary.md`.

---

## How To Regenerate

Regenerate the handoff sheet at any time from pipeline artifacts:

```powershell
# Task-sheet mode (lesson-level review approval, no per-segment QC data yet)
dart run tool/export_human_import_handoff.dart

# Intake mode (per-segment QC_PASS from completed intake)
dart run tool/export_human_import_handoff.dart `
  --intake-manifest=docs/voiceover_production_lessons_1_16/recording_intake_qc_outputs/completed_segments_report.csv
```

Both modes write to `docs/voiceover_production_lessons_1_16/` by default.
Use `--output-dir=<path>` to redirect.
