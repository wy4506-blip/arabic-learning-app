# 1. Working title

Meet `بيت`, Make It Stick

# 2. Core objective

Help the learner recognize and recall `بيت` as the Arabic word for `house`.

# 3. Why this lesson exists after Stage B

Stage B ended with the learner owning a small Arabic pack they can already read, hear, recall, and say a little: `كتاب`, `باب`, `قلم`, `هذا كتاب`, and `هذا قلم`.

Lesson 9 exists to prove that the learner can now add one more real Arabic word without feeling like they are starting from zero again. It should feel like a real new-word success first, and only quietly like the beginning of a better learning method.

This lesson should not feel like a study-skills detour or a visible four-box theory model. Its job is simpler and more product-like: the learner leaves with one real new word, `بيت`, and a growing sense that new Arabic words can be learned in a steadier, more reusable way.

# 4. Learner-visible outcome

By the end of the lesson, the learner should feel:

- "I learned a real new Arabic word: `بيت`."
- "I can recognize it when I see it or hear it."
- "I can bring it back when I mean `house`."
- "I can feel a better way to make a new Arabic word stick."

# 5. Hidden self-learning outcome

The lesson quietly teaches that a new Arabic word becomes more stable when the learner keeps together:

- how it looks
- how it sounds
- what it means
- one small note about what kind of word it is

Here, that small note is light and beginner-safe: `بيت` is a singular noun, or learner-facing, a `one-house word`.

The point is not formal analysis. The point is to make the learner feel that Arabic words can be learned as whole, reliable units.

# 6. Target knowledge scope

Primary target:

- `بيت`
- meaning: `house`
- pronunciation support: `bayt`
- simple category cue: singular noun / `one-house word`

Supporting known review anchors:

- `كتاب`
- `باب`
- `قلم`

Scope boundary:

- The lesson teaches only one new formal target: `بيت`.
- Known Stage B words may appear only as safe contrast or pack bridge items.
- Do not turn this lesson into a phrase lesson.
- If `هذا بيت` appears at all, it should be only a light bridge or optional preview, not a mastery target.
- The category cue supports memory; it must not become grammar teaching.
- If a fully vowelled support form is shown, use it only as first-pass reading support while keeping the formal target stored as `بيت`.

# 7. Input design

Input should make `بيت` feel like a real new Arabic word joining an already growing pack.

Recommended input flow:

1. Real-word arrival
   - Show `بيت` clearly, optionally with first-pass support as `بَيْت`
   - Give audio
   - Show meaning `house`
   - Optional pronunciation support: `bayt`
   - Learner-facing line: "This is one more real Arabic word you can keep."

2. Make-it-stick framing without theory language
   - Re-show `بيت` through a few small beats rather than a visible box model:
     - see the word
     - hear the word
     - connect it to `house`
     - notice one tiny note: this is a `one-house word`
   - The learner should feel guided, not lectured

3. Pack bridge
   - Briefly connect `بيت` to the learner's existing real-word pack:
     - `كتاب`
     - `باب`
     - `قلم`
     - now `بيت`
   - This should feel like the pack is growing, not like a random new card dropped in isolation

Input should avoid:

- a visible four-box framework
- grammar terminology overload
- multiple new vocabulary targets
- broad explanation about Arabic noun systems

# 8. Recognition design

Recognition should confirm that the learner can connect `بيت` across sight, sound, meaning, and one light sticky note.

Recommended recognition sequence:

1. Arabic -> meaning
   - Show `بيت`
   - Ask the learner to choose `house` from a very small set

2. Meaning -> Arabic
   - Show `house`
   - Ask the learner to choose `بيت` from a very small set including familiar decoys such as `كتاب`, `باب`, and `قلم`

3. Audio -> Arabic or meaning
   - Play audio for `بيت`
   - Ask the learner to choose the matching Arabic word or meaning

4. Light category recognition
   - Ask one very small note question such as:
     - Which note fits `بيت` in this lesson?
     - `one-house word`
     - `many houses`
     - `short line`
   - This should feel like a helpful label, not a formal grammar test

Recognition items should:

- keep option counts low
- stay anchored in the known Stage B pack
- make the new word feel more stable, not more analyzed

# 9. Recall design

Recall must prove that the learner can bring back `بيت`, not only recognize it.

Required recall-bearing moment:

1. Meaning -> Arabic retrieval
   - Prompt with `house`
   - The learner must produce or assemble `بيت`
   - This is the main evidence-bearing step

Recommended delayed retrieval:

2. Delayed word recovery after interference
   - After one or two recognition items using `كتاب`, `باب`, or `قلم`, ask for `house` again
   - The learner must retrieve `بيت` with reduced support

Optional low-stakes sticky-note check:

3. Light category recall
   - Ask whether `بيت` here is a `one-house word` or not
   - This is supportive, not the main mastery evidence

Acceptable implementation formats:

- tap-to-build `بيت`
- select-in-order assembly for `بيت`
- type-response if Arabic input is smooth in the runtime

Recall must not be satisfied by:

- choosing `بيت` only from options
- repeating it while the answer remains visible
- recalling only the English meaning without recovering the Arabic word

# 10. Output design

Output should make the new word feel owned and real.

Recommended output:

- one guided `speakResponse` or repeat-after-recognition moment for `بيت`

Purpose of output:

- reinforce that the learner is not only seeing and hearing `بيت`, but can also say it once with confidence
- create a clear new-word success moment
- keep the lesson feeling like real Arabic learning, not word analysis

Output boundary:

- do not grade fine pronunciation detail
- do not require phrase production
- keep output to one low-pressure word success action

# 11. Completion design

Completion should feel like: one more real word is now in the learner's Arabic pack.

The learner should complete the lesson only after demonstrating:

- they can recognize `بيت` from Arabic text
- they can recognize `بيت` from audio
- they can retrieve `بيت` from the meaning `house`
- they can recognize one light category cue that helps the word stick

Completion message should make the gain visible:

- the learner now owns `بيت`
- the learner's real Arabic pack is growing
- the learner is starting to feel what helps a new Arabic word stick
- the learner is ready for the next lesson, where Arabic gives a visible clue on the page

Completion should not claim:

- broad word-study mastery
- grammar understanding
- phrase mastery
- broad vocabulary growth beyond the one new word

# 12. Completion contract

## mastery_status

`mastered` only if the learner completes at least one real recall-bearing retrieval of `بيت` after initial exposure and also succeeds on basic text/audio recognition for the word.

`needs_review` if the learner can recognize `بيت` but cannot retrieve it reliably from `house`, or if they lose the word under light contrast with known pack items.

## learning_evidence

Minimum acceptable evidence:

- matched `بيت` to `house` or `house` to `بيت`
- identified `بيت` from audio among a very small set
- retrieved `بيت` from the meaning `house` in at least one recall-bearing step
- correctly recognized one light category cue for the word

Stronger evidence:

- retrieves `بيت` correctly after a short delay or after contrast with `كتاب`, `باب`, and `قلم`
- completes one low-pressure spoken output on `بيت`
- keeps the word stable across sight, sound, and meaning without heavy support

## review_seed_candidates

Priority candidates:

- `word_meaning_recall:bayt`
- `audio_to_word:bayt`
- `pack_contrast:bayt_vs_kitab_bab_qalam`

Lower-priority support seed:

- `word_note:bayt_singular_noun`

Fallback candidate if retrieval is weak:

- `supported_rebuild:bayt`

## next_home_action

If `mastered`:

- continue to Lesson 10, where the learner notices a helpful visible clue inside a very small new word set

If `needs_review`:

- schedule a short `بيت` review-first step before Lesson 10

# 13. Review seed logic

Seed 1: `word_meaning_recall:bayt`

- Trigger when the learner hesitates or fails on `house -> بيت`
- Purpose: strengthen true retrieval of the new word, not just recognition

Seed 2: `audio_to_word:bayt`

- Trigger when the learner misses or guesses on audio identification
- Purpose: stabilize the sound of `بيت` against its written form

Seed 3: `pack_contrast:bayt_vs_kitab_bab_qalam`

- Trigger when the learner confuses `بيت` with an already known Stage B word
- Purpose: help the new word enter the existing pack cleanly instead of floating as a weak isolated card

Seed 4: `word_note:bayt_singular_noun`

- Trigger when the learner loses the light category cue completely
- Purpose: restore one small sticky note that helps the word feel more learnable and memorable

Seed priority rule:

- prioritize new-word retrieval weakness first
- then audio weakness
- then pack-contrast confusion
- keep the category cue as lower-priority support only

Review boundary:

- review should still feel like re-contact with one real new word
- do not let review turn into grammar explanation or abstract analysis

# 14. Paid-boundary note

This free lesson should feel substantial because it gives the learner one real new word and the first feeling that there is a better way to hold new Arabic words in memory.

Paid expansion value remains intact because later content can still deepen:

- much broader vocabulary growth
- richer semantic groupings and word families
- more flexible phrase use around new words
- deeper pattern and morphology awareness
- repeated guided practice turning word-study into long-term skill

Free value here is: "I learned one real new Arabic word and it felt more solid."
Paid value later is: "I can grow that into a much wider, deeper, more reusable Arabic vocabulary system."

# 15. Implementation notes

- Keep exactly one formal lesson objective: recognize and recall `بيت = house`.
- The learner-facing experience should center on learning one real word, not on being taught a visible study framework.
- The form-sound-meaning-category bundle should be distributed naturally across the lesson, not shown as a heavy four-box model.
- Use `كتاب`, `باب`, and `قلم` only as safe contrast anchors from the owned pack.
- Preferred V2 flow:
  - Input
  - Recognition
  - Recognition
  - Recall
  - Output
  - Completion
- Ensure there is at least one required meaning -> Arabic recall-bearing step for `بيت`.
- If the runtime supports only lightweight production, use a tap-build or ordered assembly task for the required recall item.
- If a second recall check is possible, place it after one intervening recognition item so success feels more earned.
- Keep all choice sets at 2-3 options maximum where possible.
- If a vowelled support form appears, use it only on first exposure and store the formal target as `بيت`.
- Avoid turning `singular noun` into a separate grammar objective; it is only a light sticky note that helps the word feel more learnable.
- If `هذا بيت` appears at all, treat it as a non-target bridge only and do not count it as completion evidence.
- Completion UI should visibly mark that `بيت` has joined the learner's growing real Arabic pack.
