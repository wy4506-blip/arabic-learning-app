# 1. Working title

One Or More? Another Arabic Clue

# 2. Core objective

Help the learner notice `one` versus `more than one` in one tiny controlled Arabic set.

# 3. Why this lesson exists after Lesson 10

Lesson 10 gave the learner one encouraging feeling: Arabic can sometimes give a clue on the page. Lesson 11 keeps that product feeling alive by adding one more useful clue, not by opening a grammar block.

The learner is not being asked to study plurals in general. The learner is being shown one small readable win: in one tiny controlled pair, Arabic can show `one` and `more than one`.

This keeps the lesson practical, beginner-safe, and visibly valuable. It feels like forward motion in real Arabic, not theory.

# 4. Learner-visible outcome

By the end of the lesson, the learner should feel:

- "Arabic can show me one and more than one."
- "I can see the difference between `سيارة` and `سيارات`."
- "I can spot the same clue again in `كلمات`, but the main win is `سيارة / سيارات`."
- "Arabic is giving me another useful reading clue."

# 5. Hidden self-learning outcome

The lesson quietly teaches a healthy beginner habit: compare two related Arabic forms, notice the visible change, and connect that change to meaning.

The learner does not need a rule system here. The learner only needs one stable experience that word form can sometimes carry a useful clue.

# 6. Target knowledge scope

Lesson promise:

- This lesson gives the learner one small readable win: in a tiny controlled set, Arabic can show `one` and `more than one`.

Single objective:

- notice `one` versus `more than one` in one tiny controlled Arabic set

Primary target and main evidence carrier:

- `سيارة = one car`
- `سيارات = cars / more than one`

Secondary support only:

- `كلمة`
- `كلمات`

Secondary support confirms the clue but does not carry equal lesson weight.

Non-target anchors:

- `كتاب`
- `باب`
- `قلم`
- `بيت`

These are contrast anchors only.

Strict boundary:

- This is not a general plural lesson.
- Do not explain Arabic plural systems, irregular plurals, or broad rules.
- Do not imply transfer beyond this tiny controlled set.
- `سيارة / سيارات` is the true center of the lesson.

# 7. Input design

Input should introduce the clue through one main readable pair.

Recommended input flow:

1. Main pair reveal
   - show `سيارة` with one-car support
   - show `سيارات` with many-cars support
   - provide audio for both
   - learner-facing line: "Arabic can show one car and more than one car."

2. Main clue notice
   - compare `سيارة` and `سيارات`
   - lightly point out that the word changed and now points to `more than one`
   - keep phrasing product-like:
     - "The word is giving you a clue."
     - "This form points to more than one."

3. Secondary support confirmation
   - show `كلمة` and `كلمات`
   - use them only to confirm that the learner can notice the same kind of clue again
   - do not make this feel like a second equally heavy vocabulary target

4. Familiar anchor contrast
   - briefly keep `كتاب`, `باب`, `قلم`, and `بيت` nearby so the learner feels the new clue inside a familiar Arabic landscape

Input should avoid:

- paradigm charts
- rule-heavy grammar wording
- treating `كلمة / كلمات` as a second main lesson
- any broad claim about Arabic plurals as a system

# 8. Recognition design

Recognition should prove that the learner can see the quantity clue, with the main weight staying on `سيارة / سيارات`.

Recommended recognition sequence:

1. Main pair recognition
   - ask which of `سيارة / سيارات` means `more than one`
   - this is mandatory

2. Visual quantity match with main pair
   - match one-car versus many-cars support to `سيارة` or `سيارات`
   - keep the main pair in focus

3. Secondary support confirmation
   - ask which of `كلمة / كلمات` means `more than one`
   - this confirms the clue but should feel lighter than the main pair

4. Light contrast check
   - contrast a target item with one known anchor such as `كتاب` or `بيت`
   - ask which item in the tiny set is showing `more than one`

Recognition should keep choice sets small and should repeatedly return to `سيارة / سيارات` as the main evidence carrier.

# 9. Recall design

Recall must create real evidence that the learner can recover the `more than one` form, not only recognize it when both options are visible.

Required recall-bearing moment:

1. Main reduced-support recovery
   - show `سيار__`
   - give the meaning cue `cars`
   - require the learner to recover `ات` or the full form `سيارات`
   - this is the primary recall evidence for the lesson

Supportive confirmation:

2. Light support-pair check
   - ask which form means `words`
   - expected answer: `كلمات`
   - this stays at the level of clue confirmation, not equal recall burden

Recall must not be satisfied by:

- only showing both forms side by side the whole time
- only allowing highlighted guessing
- recalling a label such as `plural` without recovering a real Arabic form

# 10. Output design

Output should turn the clue into one simple active success moment.

Required output:

- one guided sort or placement action using `سيارة` and `سيارات`
- the learner actively places one item under `one` and the other under `more than one`

Purpose of output:

- make the learner use the distinction, not just observe it
- keep the lesson practical and readable
- reinforce that the main win lives in the primary pair

Output boundary:

- do not require explanation of grammar
- do not require free production of all forms
- do not give the support pair equal output weight

# 11. Completion design

Completion should feel like a real progress proof, not just the end of another preview.

The lesson is complete only when the learner has shown:

- `سيارات` is `more than one`
- `كلمات` is also `more than one`
- at least one reduced-support recovery of a `more than one` form, preferably `سيارات` from `سيار__`
- one active output using `one` versus `more than one`

Completion message should make the gain visible:

- Arabic can now show the learner one and more than one in a tiny set
- the learner has a new reading clue, not a memorized grammar rule
- the learner is ready to carry this clue into a tiny supported Arabic material next

Completion should not claim:

- mastery of Arabic plurals
- broad grammar knowledge
- transfer across the full language

# 12. Completion contract

## mastery_status

`mastered` only if the learner:

- identifies `سيارات` as `more than one`
- identifies `كلمات` as `more than one`
- completes at least one reduced-support recovery of a `more than one` form, preferably `سيارات` from `سيار__`
- completes one active output using `one` versus `more than one`

`needs_review` if the learner can follow the side-by-side examples but cannot recover the primary `more than one` form when support is reduced, or if the learner repeatedly loses the `one` versus `more than one` distinction in the tiny target set.

## learning_evidence

Minimum acceptable evidence:

- identified `سيارات` as `more than one`
- identified `كلمات` as `more than one`
- recovered at least one reduced-support `more than one` form, preferably `سيارات` from `سيار__`
- completed one active output using the `one` versus `more than one` distinction

Stronger evidence:

- stays stable on `سيارة / سيارات` across text, audio, and quantity support
- treats the support pair as confirming evidence without losing the main pair
- keeps the learner-facing meaning of the clue tied to quantity rather than to a memorized grammar label

## review_seed_candidates

Priority candidates:

- `one_vs_more_main_pair_sayyara`
- `recover_more_than_one_form_sayyaraat`
- `main_pair_quantity_match`

Lower-priority support candidates:

- `one_vs_more_support_pair_kalima`
- `quantity_meaning_one_vs_more`
- `target_vs_anchor_contrast`

## next_home_action

If stable, move to Lesson 12 for a tiny supported Arabic card where the learner carries forward known words plus early clue awareness.

If unstable, surface a short review on `سيارة / سيارات` first, then re-check whether the learner can still recover the `more than one` form with reduced support.

# 13. Review seed logic

Review should keep the lesson centered on one main evidence carrier.

Priority order:

1. `one_vs_more_main_pair_sayyara`
   - use when the learner confuses `سيارة` and `سيارات`
   - strongest seed because the main pair is the lesson's real target

2. `recover_more_than_one_form_sayyaraat`
   - use when the learner fails the reduced-support recovery step such as `سيار__ -> سيارات`
   - this is the most important recall seed

3. `main_pair_quantity_match`
   - use when the learner can read the forms but mismatches one-item versus many-item support

4. `one_vs_more_support_pair_kalima`
   - use only when the learner misses the confirming second example
   - this remains secondary

5. `target_vs_anchor_contrast`
   - use when the learner loses the quantity clue under light contrast with `كتاب`, `باب`, `قلم`, or `بيت`

Review should not create a generalized plural deck. It should stay tightly tied to the controlled evidence of this lesson.

# 14. Paid-boundary note

The free-course value here is strong because the learner gets one real new clue: Arabic can sometimes show `one` and `more than one` inside the word itself.

Paid expansion value remains intact because this lesson does not attempt to teach the Arabic plural system, irregular plurals, broader quantity structures, or wider transfer. Those belong later.

# 15. Implementation notes

- Keep exactly one core objective.
- Make `سيارة / سيارات` the only true main pair across input, recognition, recall, output, and completion.
- Use `كلمة / كلمات` only as secondary support that confirms the clue.
- Keep `كتاب`, `باب`, `قلم`, and `بيت` as contrast anchors only.
- Preserve the preferred V2 flow:
  - Input
  - Recognition (main pair)
  - Recognition (support or visual quantity match)
  - Recall (main reduced-support recovery)
  - Output (sort `سيارة / سيارات`)
  - Completion
- Real mastery evidence must include:
  - identifying `سيارات` as `more than one`
  - identifying `كلمات` as `more than one`
  - recovering at least one reduced-support `more than one` form, preferably `سيارات` from `سيار__`
  - completing one active output using `one` versus `more than one`
- Do not drift into broad plural-system teaching.
- Do not imply general transfer beyond this tiny controlled set.
