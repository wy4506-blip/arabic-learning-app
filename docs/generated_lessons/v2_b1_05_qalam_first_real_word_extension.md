# 1. Working title

One More Real Word: قلم

# 2. Core objective

Help the learner recognize and recall `قلم` as the Arabic word for `pen`.

# 3. Why this lesson exists after Stage A

Stage A established that the learner can already enter Arabic through real content, own two real words (`كتاب`, `باب`), notice connection/form changes inside meaningful words, and read known words with support. Lesson 5 exists to convert that early confidence into forward motion.

This lesson should not feel like a reset into abstract script work or a generic third noun drill. Its role is to show that Arabic is already becoming a small usable world: the learner can add one more real object word, recognize it by sight and sound, recall it from meaning, and feel that known Arabic content is beginning to accumulate.

It also prepares the learner for Lesson 6 by making `قلم` ready to enter a very small demonstrative frame next, without teaching that frame yet.

# 4. Learner-visible outcome

By the end of the lesson, the learner should feel:

- "I now know another real Arabic word: `قلم`."
- "I can recognize it when I see it or hear it."
- "I can bring it back when I mean `pen`."
- "My Arabic is growing into a small real set, not just isolated symbols."

# 5. Hidden self-learning outcome

The lesson quietly reinforces that real Arabic learning means holding together:

- word form
- word sound
- word meaning
- word recall

It also strengthens the learner's trust that Arabic words can be learned as meaningful whole units, while still allowing script/form awareness to ride underneath as support rather than as the main lesson content.

# 6. Target knowledge scope

Primary target:

- `قلم`
- meaning: `pen`
- pronunciation support: `qalam`
- morphology note: singular noun

Supporting known review anchors:

- `كتاب`
- `باب`

Scope boundary:

- The lesson teaches only one new formal target: `قلم`.
- `كتاب` and `باب` may appear only as familiar contrast or comparison items.
- Do not formally teach `هذا + noun` in this lesson.
- If a fully vowelled support form is shown, use it only as reading support on first exposure, not as a separate learning target.

# 7. Input design

Input should present `قلم` as a real, concrete Arabic word with immediate meaning, not as a letter exercise.

Recommended input flow:

1. First encounter
   - Show `قلم`
   - Give audio
   - Show meaning `pen`
   - Optional pronunciation support: `qalam`
   - Learner message: this is one more real Arabic word you can own

2. Meaningful reinforcement
   - Re-show `قلم` with a simple object image or object cue
   - Keep attention on whole-word recognition
   - Briefly connect it to the learner's growing known set: `كتاب`, `باب`, `قلم`

3. Forward-motion framing
   - A short learner-facing line can signal:
     "`قلم` is another everyday word you can now spot in Arabic."
   - This should create momentum without expanding into phrase teaching

Input should avoid:

- isolated letter naming
- stroke/form explanation as the main activity
- overexplaining pronunciation mechanics
- introducing additional new vocabulary targets

# 8. Recognition design

Recognition should confirm that the learner can map the new word across sight, sound, and meaning.

Recommended recognition sequence:

1. Arabic -> meaning
   - Show `قلم`
   - Ask learner to choose `pen` from a very small set

2. Meaning -> Arabic
   - Show `pen`
   - Ask learner to choose `قلم` from a very small set including familiar decoys such as `كتاب` and `باب`

3. Audio -> Arabic
   - Play audio for `قلم`
   - Ask learner to choose the matching Arabic word from 2-3 options

4. Pack recognition
   - Ask the learner to identify which of `كتاب / باب / قلم` means `pen`
   - This should feel like expanding a known set, not like random multiple-choice drilling

Recognition items should:

- keep option count low
- rely on Stage A known words as safe contrast
- make success feel earned but accessible

# 9. Recall design

Recall must produce real evidence that the learner can retrieve `قلم`, not just recognize it.

Required recall-bearing moment:

1. Meaning -> Arabic retrieval
   - Prompt with `pen`
   - Learner must produce or assemble `قلم`
   - This is the main evidence-bearing step

Recommended second recall support:

2. Delayed retrieval after interference
   - After one or two recognition items using `كتاب` and `باب`, ask again for `pen`
   - The learner must retrieve `قلم` with reduced support

Acceptable implementation formats:

- tap-to-build `قلم`
- select-in-order assembly for `قلم`
- type-response if the runtime supports Arabic input smoothly

Recall must not be satisfied by:

- only choosing `قلم` from options
- only repeating after a prompt with the answer already visible
- only audio shadowing without retrieval

# 10. Output design

Output should be small, beginner-safe, and confidence-building.

Recommended output:

- a single speak-response or repeat-after-recognition moment for `قلم`

Purpose of output:

- reinforce that the learner is not only seeing meaning but also saying a real Arabic word
- create a clear success moment without turning the lesson into pronunciation training

Output boundary:

- do not grade nuanced pronunciation
- do not require phrase production
- keep output to one low-pressure success action

# 11. Completion design

Completion should feel like proof of progress, not just the end of a preview card sequence.

The learner should complete the lesson only after demonstrating:

- they can recognize `قلم` from Arabic text
- they can recognize `قلم` from audio
- they can retrieve `قلم` from the meaning `pen`

Completion message should make the gain visible:

- the learner now owns a third real Arabic word
- their known beginner set has expanded from two real words to three
- they are ready for the next lesson's tiny usage frame

Completion should not claim:

- broad reading mastery
- phrase mastery
- general noun-system mastery

# 12. Completion contract

## mastery_status

`mastered` only if the learner completes at least one real recall-bearing retrieval of `قلم` after initial exposure and also succeeds on basic recognition.

`needs_review` if the learner can recognize `قلم` but fails the recall-bearing step or only succeeds through heavy support.

## learning_evidence

Minimum acceptable evidence:

- matched `قلم` to `pen` or `pen` to `قلم`
- identified `قلم` from audio among a very small set
- retrieved `قلم` from the meaning `pen` in at least one recall-bearing step

Stronger evidence:

- retrieves `قلم` correctly after a short delay or after contrast with `كتاب` and `باب`
- completes one low-pressure spoken output on `قلم`

## review_seed_candidates

Priority candidates:

- `word_meaning_recall:qalam`
- `audio_to_word:qalam`
- `word_contrast:qalam_vs_kitab_bab`

Fallback candidate if recall is weak:

- `supported_rebuild:qalam`

## next_home_action

If `mastered`:

- continue to Lesson 6 as the next home action, where `قلم` can enter a tiny fixed expression frame

If `needs_review`:

- schedule a short review-first exposure for `قلم` before Lesson 6

# 13. Review seed logic

Seed 1: `word_meaning_recall:qalam`

- Trigger when the learner hesitates or fails on `pen -> قلم`
- Purpose: strengthen true retrieval, not just recognition

Seed 2: `audio_to_word:qalam`

- Trigger when the learner misses or guesses on audio identification
- Purpose: connect the sound of `قلم` to the known written form

Seed 3: `word_contrast:qalam_vs_kitab_bab`

- Trigger when the learner confuses `قلم` with an already known Stage A word
- Purpose: stabilize the growing real-word set as a coherent beginner pack

Seed priority rule:

- prioritize retrieval weakness first
- then audio weakness
- then contrast confusion

Review boundary:

- review should still feel like brief re-contact with a real word
- do not turn review into isolated script remediation

# 14. Paid-boundary note

This free lesson should feel substantial because it gives the learner a clear, owned new word and makes the Arabic beginner pack visibly larger.

Paid expansion value remains intact because later content can still deepen:

- broader everyday vocabulary coverage
- richer object sets
- more varied phrase frames
- denser listening and usage practice
- stronger fluency and retention loops

Free value here is: "I learned another real Arabic word and can actually retrieve it."
Paid value later is: "I can grow that into wider usable Arabic."

# 15. Implementation notes

- Keep exactly one formal lesson objective: `recognize and recall قلم = pen`.
- Reuse `كتاب` and `باب` only as familiar contrast anchors.
- Preferred V2 flow:
  - Input
  - Recognition
  - Recognition
  - Recall
  - Output
  - Completion
- Ensure there is at least one genuine recall-bearing item that is required for completion.
- If the runtime supports only lightweight production, use a tap-build or ordered assembly task for the required recall item.
- If a second recall check is possible, place it after one intervening recognition item so the success feels more earned.
- Keep all choice sets at 2-3 options maximum.
- If a vowelled support form appears, use it only as first-pass support and keep the formal target stored as `قلم`.
- Do not introduce `هذا قلم` as a formal teaching target yet; reserve that structure for Lesson 6.
- Completion UI should visibly mark that the learner now has a 3-word real Arabic set: `كتاب`, `باب`, `قلم`.
