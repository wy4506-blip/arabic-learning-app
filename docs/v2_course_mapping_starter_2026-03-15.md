# V2 课程映射起步表（基于当前 sampleLessons）

日期：2026-03-15

## 1. 结论先行

基于当前 [lib/data/sample_lessons.dart](lib/data/sample_lessons.dart) 的 16 节 sample lesson，可先得到 4 个直接结论：

1. 当前 lesson 主体几乎全部是 Phase 3 基础表达和 Phase 4 基础语法感内容，不包含 V2 主路径所需的 Phase 1 字母与发音建立、Phase 2 拼读与识词起步。
2. 当前所有 lesson 都被统一建模为 dialogue 类，这不利于 V2 的能力导向课型划分。
3. 当前 lesson 时长集中在 18 到 23 分钟，明显高于 V2 推荐的 6 到 10 分钟核心路径，绝大多数应优先按“拆分”处理。
4. 当前内容本身大多可用，问题主要在结构、目标密度、达标规则和复习挂接，不应默认推翻内容本身。

## 2. 当前 lesson 结构性判断

### 2.1 已确认特征

- lesson 数量：16
- unit 数量：4
- lesson 时长范围：18 到 23 分钟
- 统一 category：dialogue
- 每课都已有：标题、目标、词汇、句型、对话、语法说明、练习

### 2.2 直接导向的 V2 改造判断

- P0 不是重写内容，而是补 V2 主路径缺失的 Phase 1 和 Phase 2
- P1 是把现有 16 节课从“大而全”拆成更轻的能力课
- P1 是补 lesson 完成与达标分层
- P1 是把词、句型、对话句拆成可进入 review 的 item
- P2 才是进一步优化微反馈、过渡页和轻量巩固体验

## 3. 首版映射原则

本表是起步表，不是最终排期表。

使用原则：

- 只做第一轮结构判断
- 优先确定每节旧 lesson 的迁移方式
- 优先标出哪些 lesson 必拆
- 不在这一步硬写最终页面细节

迁移方式说明：

- A：原 lesson 保留，只补标准
- B：原 lesson 拆成两个或多个 V2 lesson
- C：多个旧内容重组为一个能力导向 lesson
- D：保留原内容，新增桥接或巩固 lesson

## 5. 建议新增的 V2 骨架课

当前 sampleLessons 之外，V2 主路径至少还应新增以下骨架内容，否则无法支撑真正的“入门路径”。

### 5.1 Phase 1 必补

- 28 个字母识别与名称
- 初中末形态识别
- 易混字母对比
- 核心短元音与基础音符
- 基础发音与音形连接

### 5.2 Phase 2 必补

- 基础拼读组合
- 带音符到去音符过渡
- 高频短词识别
- 拼读到识词的桥接练习

## 6. 优先级建议

### 6.1 P0

- 建立 Phase 1 和 Phase 2 主路径
- 定义新的 lesson 状态与达标规则
- 建立 item 进入 review 的最低数据口径

### 6.2 P1

- 从当前 16 节课里先拆出最关键的一批 V2 lesson
- 第一批优先从 U1L1 到 U1L4 开始，因为这是最前置、复用最高、也最直接影响新手留存的部分

### 6.3 P2

- 再处理 U2 到 U4 的场景课
- 为高频表达和高频词汇补桥接课与巩固课

## 7. 第一批建议落地顺序

建议先做 8 到 10 节 V2 lesson 的首批试点，而不是一次性拆完 16 节旧课。

建议首批：

1. U1L1-A 问候识别
2. U1L1-B 基础回应
3. U1L2-A 名字与来自哪里
4. U1L2-B 我是学生
5. U1L3-A 课堂指令
6. U1L3-B 我不懂 / 请重复
7. U1L4-A 教室物品词
8. U1L4-B هذا / هذه
9. 一个 U1 桥接巩固课
10. 一个 U1 阶段回顾课

## 8. 对旧音频的处理建议

结合当前已有课程例句与录音工作，应采用以下策略：

- 旧词条和旧例句只要文本正确，优先保留
- lesson 拆分后优先复用原词条音频与例句音频
- 新增的 intro、过渡解释、收口提示，采用局部补录
- 不因结构拆分而默认废弃原内容音频
